# AGI Alpha SecureRails

**AI-agent security governance and proof-bound defensive remediation.**

SecureRails makes AI-agent work safe to review, safe to replay, and safe to remediate. It secures the rails for autonomous software work by converting agent actions, workflow changes, findings, and remediation proposals into replayable ProofBundles, Evidence Dockets, redacted safety ledgers, safe PRs, and reusable defensive capability — with human-governed promotion and no autonomous claim escalation.

## Product boundary

SecureRails is AI-agent security governance and proof-bound defensive remediation. It is not autonomous cybersecurity certification, not offensive cyber, not a high-risk decision system by intended purpose, not a GPAI model provider by default, and not an investment product.

## The operating chain

```text
α-AGI Insight
→ Nova-Seeds
→ MARK
→ Cybersecurity Sovereigns
→ AGI Jobs
→ ProofBundles
→ Evidence Dockets
→ CyberSecurityCapabilityArchive
→ Descendant Defensive Tasks
→ CyberSovereignPolicy-vNext
```

| Layer | Meaning |
|---|---|
| α-AGI Insight | Discovers repo-owned defensive opportunity gradients. |
| Nova-Seeds | Encode defensive variants: task, validator, replay, runbook, safe PR, evidence. |
| MARK | Allocates bounded capacity, risk tier, review priority, and validation budget proxy. |
| Cybersecurity Sovereigns | Bounded defensive proof-producing organs. |
| AGI Jobs | Execute bounded repo-owned defensive work. |
| ProofBundles | Make work replayable. |
| Evidence Dockets | Make claims reviewable. |
| CyberSecurityCapabilityArchive | Stores accepted and rejected defensive capability. |
| Descendant Defensive Tasks | Test whether capability reuse improves future defensive work. |
| CyberSovereignPolicy-vNext | Tests mechanism-level improvement; persistence requires human review. |

## What SecureRails does

- Converts agentic work events into evidence objects.
- Records redacted safety ledgers.
- Produces safe PR proposals for human review.
- Tracks ProofBundles, Evidence Dockets, hashes, replay state, cost ledgers, and safety counters.
- Preserves rejected findings and variants.
- Builds reusable defensive capability archives.
- Provides a regulatory-safe posture for defensive AI-agent governance.

## What SecureRails does not do

- It does not perform offensive cyber.
- It does not scan external targets.
- It does not execute exploits.
- It does not generate malware.
- It does not print secrets.
- It does not auto-merge.
- It does not certify that a system is secure.
- It does not evaluate workers or people.
- It does not perform profiling of natural persons.
- It does not act as a critical-infrastructure safety component by intended purpose.
- It does not claim achieved AGI, ASI, empirical SOTA, safe autonomy, official benchmark victory, or real-world certification.
- It does not offer `$AGIALPHA` as equity, debt, yield, profit rights, ownership, dividends, or appreciation.
