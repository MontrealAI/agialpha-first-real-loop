# AI-Agent Security Governance

SecureRails governs AI-agent software work by turning actions into evidence.

## Evidence conversion rule

Every agentic work event becomes one of:

1. ProofBundle;
2. Evidence Docket;
3. redacted safety ledger;
4. safe PR proposal;
5. rejected finding;
6. archived defensive capability.

## Reviewability

A reviewer must be able to answer:

- What was attempted?
- What changed?
- What proof exists?
- Can it replay?
- What did it cost?
- What risks were checked?
- Were secrets redacted?
- Was there any external target scan?
- Was there any exploit execution?
- Was there any auto-merge?
- What human decision was recorded?

## No autonomous escalation

Autonomous evidence production is permitted. Autonomous claim promotion and remediation persistence are not.

## Claim footer

No Evidence Docket, no empirical SOTA claim. Autonomous evidence production is allowed; autonomous claim promotion is not.
