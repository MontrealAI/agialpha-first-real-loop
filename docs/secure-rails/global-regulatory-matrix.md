# SecureRails Global Regulatory Matrix

This matrix is a deployment triage tool, not legal advice.

| Region / regime | Main issue | SecureRails default posture | Required control |
|---|---|---|---|
| EU AI Act | Prohibited, high-risk, GPAI, transparency | Non-high-risk governance/audit tool by default; not GPAI provider by default. | AI Act triage before customer production. |
| EU GDPR | Personal data, automated decisions | Avoid person-level profiling and solely automated significant decisions. | Data minimization, DPA/DPIA where needed. |
| EU CRA | Software product with digital elements | Treat SecureRails as software requiring secure development and vulnerability handling if placed on EU market. | Secure SDLC, vulnerability reporting plan, user instructions. |
| EU NIS2 | Essential/important entities | Customers may need supplier controls. SecureRails can support governance evidence. | Vendor security pack and incident process. |
| EU MiCA | Crypto-assets and CASP activity | `$AGIALPHA` utility-only; no investment, yield, profit, ownership, or appreciation language. | Token utility review and counsel before offers/listings. |
| United States | FTC, SEC, state privacy, cybersecurity representations | Avoid deceptive AI/security claims and investment framing. | Marketing guardrail, token boundary, security claims review. |
| Canada | Privacy, consumer protection, AI governance | Avoid high-impact/person decisions by default. | Deployment intake and privacy review. |
| United Kingdom | AI assurance, data protection, financial promotions | Governance-support posture; no investment token promotion. | Claims review and financial-promotion review. |
| Singapore | AI governance, data protection, digital payment token regimes | Utility-only token boundary; defensive governance posture. | Deployment intake, privacy/security review. |
| Japan | AI governance, privacy, crypto | Defensive support; no investment framing. | Token and privacy review. |
| UAE / GCC | AI/cyber/data/local cloud rules | Enterprise deployment controls. | Local counsel and hosting/security review. |

## Global rule

Do not sell SecureRails as a replacement for legal review, security certification, conformity assessment, penetration testing, SOC 2, ISO 27001, or regulatory approval. Sell it as evidence infrastructure and human-governed defensive remediation.
