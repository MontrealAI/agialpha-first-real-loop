# Security Advisory Process

Use GitHub repository security advisories for validated repository vulnerabilities requiring coordinated remediation. Keep reports private while triage/fix is active.

Checklist:
1. Receive report.
2. Triage severity.
3. Confirm scope.
4. Create incident record.
5. Open private advisory if appropriate.
6. Build fix.
7. Validate fix.
8. Generate Evidence Docket.
9. Prepare disclosure.
10. Publish only when ready.

Request CVE only when materially appropriate. Avoid overclaim.
