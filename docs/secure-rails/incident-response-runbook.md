# Incident Response Runbook

Phases: Prepare, Detect, Triage, Contain, Eradicate, Recover, Communicate, Evidence Docket, Lessons Learned, Capability Archive update.

Incident classes include claim-boundary failure, auto-merge enabled, raw secret printed, external target scan attempted, exploit execution attempted, malware-like output, social-engineering content, unsafe customer artifact, workflow permission drift, Evidence Mission Control publication failure, $AGIALPHA utility-boundary violation, EU AI Act posture drift, GPAI-provider posture drift.

Required incident record fields: incident_id, detected_at, severity, affected_scope, safety_counters, evidence, containment, human_reviewer, status, claim_boundary, lessons_learned, archive_update.
