# GPAI Dependency Boundary

## Policy

SecureRails is not a GPAI model provider by default. It is an AI-agent security governance, evidence, replay, redaction, and human-review orchestration layer.

## What SecureRails provides

- Workflow governance.
- Evidence Dockets.
- ProofBundles.
- Safety ledgers.
- Safe PR proposal workflows.
- Claim-boundary controls.
- Reusable defensive capability archives.

## What SecureRails does not provide by default

- A general-purpose AI model placed on the market.
- Model training services.
- Model weights.
- A foundation model API under SecureRails branding.
- A claim that SecureRails is a general-purpose model.

## Third-party model usage

If SecureRails uses third-party models, those model providers remain responsible for their own model-provider obligations. SecureRails customers and operators must maintain a model dependency register.

## Trigger for re-screening

If SecureRails later ships, fine-tunes, distributes, or places on the market its own general-purpose model, a separate GPAI-provider review is mandatory before release.

## Customer disclosure

> SecureRails may integrate with third-party AI systems or models. SecureRails does not by default provide a general-purpose AI model; it provides governance, evidence, replay, redaction, and human-review controls for agentic software work.

## Claim footer

No Evidence Docket, no empirical SOTA claim. Autonomous evidence production is allowed; autonomous claim promotion is not.
