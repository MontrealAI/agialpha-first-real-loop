# AGI ALPHA PHOENIX-HUB-001

**Experiment:** Self-Healing Evidence Institution Gauntlet

PHOENIX-HUB-001 is designed to impress a skeptical reviewer by testing whether AGI ALPHA can improve the institution that proves AGI ALPHA.

It scans the repository, discovers workflows/pages/dockets/challenge packs, identifies publication and evidence-registry regressions, compares B0-B7 institutional-recovery baselines, generates a persistent-registry preview, emits Evidence Dockets, creates a clean site preview, produces safe PR materials, and supports external reviewer challenge packs.

## Why this matters

The AGI ALPHA paper says empirical claims require Evidence Dockets with real tasks, baselines, ProofBundles, replay logs, cost/safety ledgers, validator reports, delayed outcomes, and independent reproduction. PHOENIX-HUB-001 makes the Evidence Hub itself a target of governed machine labor: the proof institution becomes the object being repaired, replayed, audited, and improved.

## Files to upload

Upload these folders/files to the repository root:

```text
agialpha_phoenix_gauntlet/
config/
tests/
phoenix_challenge_packs/
COPY_WORKFLOWS/
COPY_ISSUE_TEMPLATES/
README_PHOENIX_HUB_001.md
```

Then upload the workflow files from `COPY_WORKFLOWS/` into `.github/workflows/`.

Then upload the issue templates from `COPY_ISSUE_TEMPLATES/` into `.github/ISSUE_TEMPLATE/`.

## Main workflow

Run:

```text
Actions → AGI ALPHA PHOENIX-HUB-001 / Autonomous → Run workflow
```
