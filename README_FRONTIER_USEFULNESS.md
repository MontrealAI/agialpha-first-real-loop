# AGI ALPHA Frontier Usefulness Experiment v0.1

This package pushes the current evidence pipeline from local L4-L7 scoring toward a usefulness frontier: not only can the system produce Evidence Dockets, it can generate concrete capacity-building artifacts that improve the evidence institution itself.

## What it does

- Scans the repository for evidence surfaces, workflows, dockets, claim boundaries, and overclaim risk.
- Generates a Frontier Usefulness Portfolio with six task dockets.
- Produces patch proposals, paper-update insert, benchmark-adapter templates, external reviewer kit, and falsification audit.
- Runs local baseline-comparative scoring with a conservative claim boundary.
- Publishes a GitHub Pages scoreboard.

## What it does not claim

It does not claim achieved AGI, ASI, empirical SOTA, safe autonomy, standard-setting control, guaranteed economic return, or civilization-scale capability. External benchmark performance requires configured and executed adapters for SWE-bench Verified, GAIA, BrowserGym/WebArena/WorkArena, OSWorld, tau-bench-style tasks, and external review.

## Workflows

Create these files in `.github/workflows/` from `COPY_WORKFLOWS/`:

- `frontier-usefulness-autonomous.yml`
- `frontier-external-review.yml`

Then run:

`Actions -> AGI ALPHA Frontier Usefulness / Autonomous -> Run workflow`

## Publication-safe public wording

AGI ALPHA now has a Frontier Usefulness Experiment that generates usefulness-focused Evidence Dockets, local baseline comparisons, external benchmark adapter templates, an external reviewer kit, and a falsification audit, while preserving the claim boundary that external benchmarks and outside replay are required for stronger claims.
